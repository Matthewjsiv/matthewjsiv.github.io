Projects

Person-tracking for Tello drone

I wrote a python script for the Tello drone so it can track and follow you in order to keep you in the frame. It uses a python port of the posenet model I use in my pose-evaluator demo (which you can see if you scroll down) to predict key points on your body. It then uses the relationships between the different points to calculate an error value which it feeds into basic PID controllers (one for each type of movement). This allows it to rotate, move up/down, and move forward/backward as it follows you. It clearly isn't perfect yet, but I am currently working on expanding it by fine tuning the PID and implementing other new features such as different tracking behaviors, as well as rebuilding it using ROS. You can see the code and more info here.

Note: if the embedded video isn't working, press the pop-out button and it'll open in a new tab.



Evaluation of Subject Viewability Using Posenet

As a proof-of-concept for a potential senior design project, I made a demo that uses your webcam to tell you how well it can see you based on key parts of your body. It uses Posenet to extract keypoints from the camera feed, and then uses various weights for different features of your overall pose. Motion-based Object Tracking

I was curious about how accurately you could track a moving object only by using the difference between frames. So, I put together this mini project in Matlab that can track moving objects relatively well. It takes your webcam feed as an input and then finds the difference in intensity of the current frame and the previous frame. This method sometimes results in only the edges of objects being visible, so it then attempts to automatically fill in more of the moving object. This end result is converted to a binary representation (shown on the right), on which I use Matlab's "blob analysis" to predict where any moving objects might be (shown on the left). My binary representation isn't as good as I'd like it to be yet, so I plan on optimizing it more, as well as maybe making the demo more interactive (for example using it to let you draw on the screen). You can look at the code here. If the embedded video isn't working, press the pop-out button and it'll open in a new tab.

Studious (still in progress)

Studious is an app for Pitt students designed to save time by showing which study spots on campus are not crowded. So far, it shows your current location, the study spots on campus, and whether those study spots are open or closed. My colleague, Andrew Ashley, and I will be working with Dr. Vorp (associate dean of engineering at Pitt) to find a way to keep track of these study spaces in real time and add this functionality to the app.
